# 秋招投递管理

一款本地运行的秋招投递状态管理工具。无需后端服务，记录保存在浏览器本地存储中。

## 功能

- 添加、查看、编辑、删除投递记录
- 底部模块导航：总览、投递、添加、提醒、我的，使用方式更接近手机 App
- 总览集中展示状态入口、筛选条件和排序后的全部记录，筛选或排序后不需要再跳到投递模块
- 投递模块只保留状态看板，用于按阶段拖拽、快速流转和查看各板块进度
- 本地账号空间，不同账号的数据默认隔离，点击左上角“秋”或底部“我的”进入账号管理
- 本地账号密码，账号可选择设置密码，切换已设密码的账号时需要验证
- 六状态流转：待初筛、待测评、待笔试、待面试、已拒绝、offer
- 搜索公司、岗位、来源和备注
- 按状态、城市筛选，按更新时间或投递时间排序
- 城市字段和城市筛选，单条记录最多选择 3 个二线及以上城市
- 记录来源类型，默认官网，支持公众号、牛客、Boss、实习僧和自定义网址
- 每次更新会写入历史记录
- 自动记录进入待测评和已拒绝的时间，并计算推进耗时、拒绝耗时
- 拒绝后可标记为后续还能再次投递，并在看板和统计中单独展示
- 流转到待测评、待笔试、待面试时强制设置截止时间，截止前 12 小时内打开 App 会强提醒
- APK 内设置截止时间后可一键打开系统日历，创建截止前 12 小时的外部提醒
- 设置下次检查日期，提醒模块展示今日待检查；点击已检查后会询问状态是否变化
- 今日待检查中选择“状态没变”会把“更新于”改为当天并顺延下次检查；选择待测评、待笔试或待面试会同步移动到看板对应板块
- JSON 导入 / 导出备份
- 重要程度 1-5 星，星级越高卡片越偏红、越醒目
- 重要提醒，支持无提醒、指定截止日期时间、从现在起 N 小时，并可填写提醒备注
- 重要提醒进入截止前 24 小时后，打开 App 会弹出强提醒窗口
- 看板和列表默认按重要程度从高到低排列
- 可设置“更新于”超过 N 个月的逾期预警，逾期记录高红显示并排在结果最后
- 统计视图：状态漏斗、面试推进率、高重要待推进、来源分布、星级分布
- 支持手机浏览器打开和添加到主屏幕
- 支持离线缓存，作为 PWA 使用

## 使用方式

直接打开 `index.html` 即可使用。

也可以在当前目录启动一个本地服务器：

```bash
python3 -m http.server 5173
```

然后访问：

```text
http://localhost:5173
```

## 手机上打开

电脑和手机连接同一个 Wi-Fi，然后在当前目录启动服务：

```bash
python3 -m http.server 5173 --bind 0.0.0.0
```

查看电脑的局域网 IP：

```bash
ipconfig getifaddr en0
```

假设输出是 `192.168.1.8`，手机浏览器打开：

```text
http://192.168.1.8:5173
```

打开后可以添加到手机主屏幕：

- iPhone：Safari 分享按钮 -> 添加到主屏幕
- Android：浏览器菜单 -> 安装应用 / 添加到主屏幕

## 数据说明

数据保存在浏览器的 `localStorage` 中，key 为：

```text
campus-application-tracker:accounts:v1
campus-application-tracker:active-account:v1
campus-application-tracker:records:v1:{accountId}
```

旧版数据会自动迁移到 `默认账号`。如果要换电脑或清理浏览器数据，请先在对应账号下点击右上角导出按钮备份 JSON。

注意：当前版本是纯本地保存。电脑浏览器和手机 APK 的数据彼此独立，不会自动同步。

双向导入导出方式：

- 电脑到手机：切到目标账号，电脑网页版点导出，得到 JSON 文件；把文件发到手机；手机 APK 切到目标账号后点导入。
- 手机到电脑：切到目标账号，手机 APK 点导出，选择系统分享、保存 JSON 文件或复制 JSON；把 JSON 文件发到电脑；电脑网页版切到目标账号后点导入。

如果后续需要真正自动同步，可以加一个本地后端、Supabase、Firebase 或 iCloud/坚果云一类的同步层。
