# 秋招投递管理

一款以本地保存为主的秋招投递状态管理工具。记录默认保存在浏览器本地存储中，也支持通过腾讯云 CloudBase 做短链接备份和账号云同步。

当前版本：`3.0.0 / versionCode 47`

## 功能

- 添加、查看、编辑、删除投递记录
- 底部模块导航：总览、投递、添加、提醒、我的，使用方式更接近手机 App
- 总览默认只展示状态入口和筛选条件，点击状态、城市、排序或搜索后，在总览下方直接展示平铺投递列表
- 默认以“本地游客”进入；“我的”页默认只展示身份卡片，可通过“去登录”和“账号管理”进入对应二级面板
- 本地账号密码，账号可选择设置密码，切换已设密码的账号时需要验证
- 六状态流转：待初筛、待测评、待笔试、待面试、已拒绝、offer
- 搜索公司、岗位、城市和备注
- 按状态、城市筛选，按更新时间或投递时间排序
- 城市字段和城市筛选，单条记录最多选择 3 个城市，支持“全国任意城市”
- 新增记录城市按“全国任意城市、杭州、北京、深圳、一线城市、二线城市拼音”排序
- 新增记录默认投递日期和更新时间为当天，下次检查默认 7 天后
- 每次更新会写入历史记录
- 自动记录进入待测评和已拒绝的时间，并计算推进耗时、拒绝耗时
- 拒绝后可标记为后续还能再次投递，并在看板和统计中单独展示
- 流转到待测评、待笔试、待面试时强制设置截止时间，截止前 12 小时内打开 App 会强提醒
- APK 内设置截止时间后可一键打开系统日历，创建截止前 12 小时的外部提醒
- 设置下次检查日期，提醒模块展示今日待检查，点击已检查后可同步更新状态或更新时间
- JSON 导入 / 导出备份
- 导入框支持短链接、旧版原始长链接，以及直接粘贴原始 JSON 内容
- 腾讯云 CloudBase 短链接导入导出：导出时显示生成进度，云端异常时自动超时并提供本地备份链接，导入时可粘贴云端短链接自动解析
- 自动云备份中心：打开 App 后距离上次备份超过 12 小时会自动上传腾讯云，保留最近 30 条备份记录，可复制链接或一键恢复
- 账号云同步：云端会分别保存账号名索引和密码校验值，新设备登录同名同密码账号后，会自动检测并询问是否恢复云端最新备份；同名但密码不同会提示密码错误
- 重要程度 1-5 星，星级越高卡片越偏红、越醒目
- 重要提醒，支持无提醒、指定截止日期时间、从现在起 N 小时，并可填写提醒备注
- 重要提醒进入截止前 24 小时后，打开 App 会弹出强提醒窗口
- 看板和总览列表默认按重要程度从高到低排列
- 可设置“更新于”超过 N 个月的逾期预警，逾期记录高红显示并排在结果最后
- 统计视图：两列紧凑指标、状态流程漏斗、城市气泡分布、星级重点池环形图、高重要待推进
- 支持手机浏览器打开和添加到主屏幕
- 支持离线缓存，作为 PWA 使用

## 使用方式

直接打开 `index.html` 即可使用。

也可以在当前目录启动一个本地服务器：

```bash
python3 -m http.server 5173
```

然后访问：

```text
http://localhost:5173
```

## 手机上打开

电脑和手机连接同一个 Wi-Fi，然后在当前目录启动服务：

```bash
python3 -m http.server 5173 --bind 0.0.0.0
```

查看电脑的局域网 IP：

```bash
ipconfig getifaddr en0
```

假设输出是 `192.168.1.8`，手机浏览器打开：

```text
http://192.168.1.8:5173
```

打开后可以添加到手机主屏幕：

- iPhone：Safari 分享按钮 -> 添加到主屏幕
- Android：浏览器菜单 -> 安装应用 / 添加到主屏幕

## 数据说明

数据保存在浏览器的 `localStorage` 中，key 为：

```text
campus-application-tracker:accounts:v1
campus-application-tracker:active-account:v1
campus-application-tracker:records:v1:{accountId}
```

旧版数据会自动迁移到 `本地游客`。如果要换电脑或清理浏览器数据，请先在对应账号下点击右上角导出按钮备份 JSON。

注意：当前版本仍以本地数据为主。开启账号云同步后，同名同密码账号可以从腾讯云恢复最近备份；它不是强实时同步，适合作为跨设备恢复和定时备份使用。

双向导入导出方式：

- 电脑到手机：切到目标账号，电脑网页版点导出，得到 JSON 文件；把文件发到手机；手机 APK 切到目标账号后点导入。
- 手机到电脑：切到目标账号，手机 APK 点导出，选择系统分享、保存 JSON 文件或复制 JSON；把 JSON 文件发到电脑；电脑网页版切到目标账号后点导入。

如果后续需要强实时同步，可以继续升级为正式后端数据库和登录体系。
