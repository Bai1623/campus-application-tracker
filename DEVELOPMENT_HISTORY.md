# 秋招投递管理开发历史

本文档记录「秋招投递管理」从最初想法到当前 APK 版本的主要开发过程。整理依据包括项目对话需求、实际功能演进和 Git 提交历史。

当前版本：`3.0.0 / versionCode 47`

`2.x` 阶段从重要提醒能力开始。
`3.x` 阶段从「我的」模块统计可视化升级开始。

## 迭代记录

### 01. 产品设想与核心设计

- 明确项目目标：做一款个人秋招投递状态管理软件，替代零散笔记。
- 确定核心记录字段：公司、来源、更新时间、状态、备注。
- 确定核心状态：待初筛、待面试、已拒绝。
- 初步确定桌面端优先，界面需要美观、顺滑、有工作台感。

### 02. 静态 HTML 原型

- 搭建第一版 `HTML / CSS / JS` 单页应用。
- 实现添加记录、查看记录、修改记录、删除记录。
- 使用浏览器 `localStorage` 做本地保存。
- 做出初版看板、列表和筛选体验。

### 03. 记录详情查看能力

- 修复添加记录后只能看到列表、不能查看每条详情的问题。
- 在统计卡片和状态入口中增加进入详情/列表的能力。
- 让用户可以从总投递、待初筛、待面试等入口查看对应记录。

### 04. Android APK 初版

- 将 Web 应用封装为 Android APK。
- 使用 Android WebView 运行本地前端资源。
- 形成可直接安装使用的移动端应用。
- 固定 APK 输出路径，方便后续反复安装测试。

### 05. 移动端导入导出

- 增加 JSON 导出能力，支持把本地记录备份出来。
- 增加 JSON 导入能力，支持手机和电脑之间迁移数据。
- 在 Android 端接入系统分享面板，方便分享到微信、文件、网盘等位置。
- 支持保存 `.json` 文件到手机本地。

### 06. 字段精简与重要程度

- 删除不必要的「子企业部门」字段。
- 删除「公众号名称」字段。
- 增加「重要程度」字段，支持 1-5 星。
- 根据重要程度调整卡片颜色：星级越高越偏红、越醒目；星级越低越偏绿色。

### 07. 状态统计与详情入口

- 强化「秋招进度工作台」下方各状态统计。
- 支持点击统计项后进入对应记录列表。
- 改善看板和列表之间的跳转体验。

### 08. 排序与统计视图

- 让看板和列表默认按重要程度从高到低排序。
- 增加统计视图，不再只有看板和列表。
- 统计中加入状态漏斗、来源分布、星级分布、高重要记录等分析模块。
- 让统计视图更贴近秋招复盘和推进节奏。

### 09. 本地账号隔离

- 增加本地账号功能。
- 不同账号默认隔离存储各自记录。
- 支持切换账号，避免多个身份或不同投递批次混在一起。
- 账号数据仍然保存在本机浏览器或 APK 本地存储中。

### 10. 上线为网页版 PWA

- 明确最简单上线方案：先作为静态网页/PWA 发布。
- 支持 GitHub Pages / Gitee Pages / Cloudflare Pages 等静态托管方式。
- 确认静态版本的限制：不同设备之间不会自动云同步。
- 完成 Git 仓库关联和远程推送流程。

### 11. 账号侧栏体验

- 将账号入口设计为点击左上角「秋」图标展开。
- 侧栏中展示当前账号、账号列表和账号管理操作。
- 避免使用 Android WebView 默认系统弹窗。
- 新增账号、重命名账号、删除确认等操作改为应用内界面。

### 12. 账号密码与管理密码

- 为账号增加本地密码保护。
- 切换已设密码账号时需要输入密码。
- 增加管理密码机制：通过管理密码后才能重置任意账号密码。
- 不允许查看旧密码，只允许重置，避免明文暴露。

### 13. 逾期预警机制

- 增加「更新时间超过 N 个月」的逾期预警。
- N 可以在界面中手动调整。
- 逾期记录高红色显示，用于提醒长期未更新的投递。
- 逾期记录在看板和列表中排在最后，内部再按星级排序。
- 统计视图中增加逾期数量栏目。

### 14. 今日待检查

- 增加「今日待检查」模块。
- 基于每条记录的下次检查日期筛选需要回看的投递。
- 增加「已检查」操作，点击后该记录会从今日待检查中移除。
- 已检查后下次检查时间顺延，减少重复提醒。

### 15. 城市字段与城市筛选

- 增加「城市」字段。
- 城市选择范围覆盖全国二线及以上城市。
- 杭州、北京、深圳、广州置于城市列表最前面。
- 其余城市按拼音顺序排列。
- 统计中增加城市筛选能力。

### 16. 多城市记录

- 将城市字段从单选升级为多选。
- 每条记录最多可选择 3 个城市。
- 城市筛选支持“包含该城市”的匹配逻辑。
- 修复城市筛选只能显示「全部城市」的问题。

### 17. 搜索跳转优化

- 搜索框输入后不再需要手动滑动到结果区域。
- 点击搜索图标或按回车后，自动跳转到符合条件的列表。
- 如果没有匹配结果，提示当前暂无该信息。
- 移除不美观的城市偏好主页面展示。

### 18. 作者标识与开源信息

- 在页面底部增加 `bai@copyright`。
- 增加「联系作者」入口。
- 点击后弹出作者信息弹窗。
- 展示开发者、联系邮箱和开源地址。

### 19. 自定义 APK 图标

- 设计并生成项目专属图标。
- 图标融合「秋」「招」「BAI」「投递卡片」「进度看板」等元素。
- 优化为偏苹果风格的毛玻璃、扁平化视觉。
- 替换 Android 默认图标。

### 20. 链接式导入导出

- 在 JSON 文件导入导出之外，增加链接式导入导出。
- 导出时可复制导入链接，方便通过微信、QQ、电脑浏览器传递。
- 导入时可以选择 JSON 文件，也可以手动粘贴链接。
- 取消自动检测 URL 导入，避免换账号或打开页面时莫名弹窗。

### 21. 云端短链接分享

- 接入 Cloudflare Workers + KV 免费云端方案。
- 导出时优先把当前账号数据上传到云端。
- 云端返回短链接，例如 `https://bai.a1623444091.workers.dev/i/Ab3x9K`；2.2.3 起切换为腾讯云 CloudBase HTTP 网关链接。
- 导入时粘贴短链接，应用从云端读取 JSON 并导入。
- 如果云端不可用，保留本地长链接作为兜底。

### 22. 账号列表与记录卡片交互

- 多账号列表改为横向网格排列，一行从左到右排满后再换行。
- 新建账号时密码改为可选。
- 看板和列表中的记录都改成卡片式交互。
- 任意记录左滑可露出「置顶」和「删除」操作。
- 置顶记录优先展示；逾期预警记录仍按既定规则排在最后。

### 23. 管理员账户删除账号

- 取消当前账号区域的「删除当前账号」按钮。
- 当前账号区域恢复为两行：重命名账号、设置/更改账号密码。
- 将「本机密码管理」改名为「管理员账户」。
- 输入管理密码后，不仅可以重置所有账号密码，也可以删除任意账号。
- 删除账号需要二次确认，并且至少保留一个账号。

### 25. 2.0.0 重要提醒

- 版本号正式进入 `2.0.0 / versionCode 26`。
- 每条记录新增必填「重要提醒」字段。
- 重要提醒支持三种方式：无、选择具体截止日期时间、从当前时间开始计算未来 N 小时。
- 支持填写提醒备注，例如在线测评、笔试、确认面试时间等。
- 当重要提醒进入截止前 24 小时且尚未过期时，每次进入 App 都会弹出强提醒窗口。
- 强提醒窗口集中展示当前账号下需要处理的重要事项，并可点击进入对应记录详情。

### 26. 2.0.1 状态扩展与版本信息

- 状态体系扩展为 6 个：待初筛、待测评、待笔试、待面试、已拒绝、offer。
- 新增记录时，当前状态只展示待初筛和待笔试，减少误选。
- 编辑已有记录时，可以选择全部 6 个状态。
- 看板和列表卡片的快速流转区固定展示除待初筛外的 5 个状态，方便快速推进。
- 底部 `bai@copyright` 旁新增当前版本号和更新时间。
- 重要提醒默认值调整为「选择截止日期时间」，减少被忽略的概率。

### 27. 2.0.2 首页与看板布局优化

- 移除主页侧边栏的来源类型筛选，减少低频筛选项占用空间。
- 看板状态顺序调整为：待测评、待笔试、待面试、待初筛、offer、已拒绝。
- 工作台统计改为一行两列，更适合快速扫读。
- 首页统计移除本周新增和超过 7 天未更新，只保留更关键的总量、状态和逾期预警。

### 28. 2.0.3 状态耗时与再次投递

- 点击流转到「待测评」时记录推进时间，并计算从创建记录到进入待测评的推进耗时。
- 点击流转到「已拒绝」时记录拒绝时间，并计算从创建记录到被拒绝的拒绝耗时。
- 详情页展示推进耗时、拒绝耗时和是否可再次投递，方便后续统计分析。
- 拒绝后支持标记「后续是否还能再次投递」。
- 新增「可再次投递」统计入口和看板专栏，放在已拒绝之后，记录公司名称、拒绝时间和上次投递岗位。

### 29. 2.0.4 拒绝流转与截止强提醒

- 已拒绝记录的快速流转区新增「待初筛」，方便误点拒绝后恢复到初筛模块。
- 已拒绝记录移除无意义的「已拒绝」快速按钮，并新增「可再次投递」快捷按钮。
- 拒绝后的再次投递选择改为应用内弹窗，不再使用浏览器原生确认框。
- 快速流转到待测评、待笔试、待面试时，必须设置截止时间后才能保存。
- 截止提醒窗口调整为截止前 12 小时触发，并放大强提醒弹窗，打开 App 时更容易看到。

### 30. 2.0.5 系统日历提醒

- 流转到待测评、待笔试、待面试并设置截止时间后，新增是否创建系统日历提醒的确认窗口。
- 选择「是」时，APK 会打开手机系统日历的新建事件页面。
- 日历事件时间自动设为截止日期前 12 小时，并带入公司、岗位、状态、截止时间和提醒备注。
- 选择「否」时，只保留 App 内强提醒，不创建系统日历事件。

### 31. 2.1.0 模块化底部导航重构

- 页面结构从原来的从上到下长网页，重构为更接近手机 App 的模块化布局。
- 底部新增 5 个入口：总览、投递、添加、提醒、我的。
- 「总览」只展示状态筛选和筛选条件，默认不展示具体投递记录；点击状态或统计后再进入结果列表。
- 「投递」保留原有看板、列表、卡片左滑、快速流转、编辑、删除、置顶等核心操作。
- 「提醒」集中展示今日待检查和截止前 12 小时的重要事项。
- 「我的」集中放置账号管理、管理员账户、数据导入导出、作者信息。
- 顶部搜索保留全局能力，搜索命中后自动进入投递列表；无结果时提示当前暂无该信息。

### 32. 2.1.1 云端短链接导入导出修复

- 修复模块化后点击导出时云端请求长时间无反馈、短链接看起来无法生成的问题。
- 点击导出后立即打开导出面板，并显示云端短链接生成进度和最终结果。
- 云端上传与读取增加 8 秒超时，避免 `workers.dev` 网络不可用时界面一直等待。
- 兼容云端返回 `url`、`shortUrl`、`id` 或 `key` 字段，并校验返回链接可被当前导入逻辑识别。
- 云端不可用时继续生成并复制本地备份链接，同时明确提示当前复制的不是云端短链接。
- 按紧急修复规则升级为 `2.1.1 / versionCode 33`，并同步提升 PWA 缓存版本。

### 33. 2.2.0 Android 安装版本升级

- 同步提升 PWA 缓存版本，避免继续加载旧版前端资源。

### 34. 2.2.1 总览列表与检查流转

- 投递模块移除列表 Tab，只保留按状态分组的看板工作区。
- 列表能力移动到总览模块：点击状态筛选、城市筛选、排序条件或搜索后，会在总览下方直接展示平铺记录。
- 总览列表沿用原有记录卡片、详情打开、快速流转、置顶和删除能力。
- 今日待检查点击「已检查」后，先询问状态是否已变更。
- 状态未变更时，会把当前记录的「更新于」改为检查当天，并顺延下次检查时间。
- 状态已变更时，可选择待测评、待笔试、待面试，并同步移动到投递看板对应板块；这些状态仍会强制设置截止时间。
- 版本升级为 `2.2.1 / versionCode 36`，保证可以覆盖安装 `2.2.0 / versionCode 35`。

### 35. 2.2.2 新增记录轻量化

- 新增记录的城市选项增加「全国任意城市」，并放在城市列表第一位。
- 城市排序调整为：全国任意城市、杭州、北京、深圳、剩余一线城市、二线城市按拼音排序。
- 新增/编辑记录表单移除来源类型和来源详情，旧数据导入时仍保留内部默认来源兼容。
- 新增记录时投递日期和更新时间默认当天，下次检查默认当天 7 天后。
- 「我的」模块移除重复的「账号统计 / 当前账号概览」卡片，统计入口集中保留在分析区域。
- 版本升级为 `2.2.2 / versionCode 37`，保证可以覆盖安装 `2.2.1 / versionCode 36`。

### 36. 2.2.3 腾讯云短链接切换

- 云端短链接服务从 Cloudflare Worker 切换到腾讯云 CloudBase HTTP 网关，国内手机网络不再依赖 VPN。
- 导出上传改为 `POST /share`，导入读取改为 `GET /share?id=短链接ID`。
- 兼容 CloudBase 云函数返回的 `payload` / `data` 数据结构，并继续保留云端失败时的本地备份链接兜底。
- 版本升级为 `2.2.3 / versionCode 38`，保证可以覆盖安装 `2.2.2 / versionCode 37`。

### 37. 2.2.4 自动云备份中心

- 打开 App 后会自动检查当前账号距离上次云备份是否超过 12 小时。
- 超过 12 小时且当前账号有投递记录时，自动上传到腾讯云 CloudBase 短链接服务。
- 「我的」模块新增「云备份中心」，可查看最近备份时间、手动立即备份、复制备份链接和恢复备份。
- 每个账号本地保留最近 30 条云备份记录，超过数量后自动裁剪旧记录。
- 继续复用 2.2.3 的腾讯云短链接接口，不需要新增后端服务。
- 版本升级为 `2.2.4 / versionCode 39`，保证可以覆盖安装 `2.2.3 / versionCode 38`。

### 38. 2.2.5 账号云同步

- 在「云备份中心」新增账号云同步能力，每个本地账号可单独设置云同步码。
- App 使用账号名和云同步码生成同步身份，不会把云同步码明文上传到云端。
- 自动/手动云备份成功后，如果当前账号已开启云同步，会把最新备份短链接登记到腾讯云同步索引。
- 新设备新增同名账号并输入相同云同步码时，会自动检查云端是否已有最新备份，并询问是否恢复。
- 新增腾讯云 CloudBase 云函数代码文件 `cloudbase/share/index.js`，支持 `sync-put` 和 `sync-get`。
- 版本升级为 `2.2.5 / versionCode 40`，保证可以覆盖安装 `2.2.4 / versionCode 39`。

### 39. 2.2.6 云端短链接自检

- 修复云端返回短链接 ID 后，如果该 ID 立刻无法读取，App 仍然复制坏短链接的问题。
- 导出云端短链接时新增读回自检，确认短链接可恢复为投递记录后才复制给用户。
- 如果腾讯云函数未部署或读写不一致，会自动走本地长备份链接兜底，避免用户拿到不可导入的短链接。
- 版本升级为 `2.2.6 / versionCode 41`，保证可以覆盖安装 `2.2.5 / versionCode 40`。

### 40. 2.2.7 原始长链接导入兼容

- 导入入口从“短链接导入”扩展为短链接、旧版原始长链接和原始 JSON 三合一导入。
- 支持直接粘贴旧版 `#import=...` 长链接，不需要先保存为 JSON 文件。
- 支持直接粘贴导出的 JSON 内容或单条记录 JSON 片段，方便恢复早期备份文本。
- 导入失败提示改得更准确，不再把所有失败都说成“已过期”。
- 版本升级为 `2.2.7 / versionCode 42`，保证可以覆盖安装 `2.2.6 / versionCode 41`。

### 41. 2.2.8 账号密码云同步登录

- 默认账号改为“本地游客”，游客模式只保存本机数据，不会主动触发云端同步。
- 将账号入口从“新增账号”调整为“登录账号”，登录时必须输入账号名和账号密码。
- 取消单独的云同步码，改为使用账号名和账号密码生成云端同步身份。
- 新设备只有输入相同账号名和相同账号密码，才能检测到并恢复该账号的云端最新备份。
- 登录成功后自动检查云端备份；有备份时弹窗展示记录数量并询问是否恢复，没有备份时提示“云端暂无数据。现在开始秋招吧！”。
- 版本升级为 `2.2.8 / versionCode 43`，保证可以覆盖安装 `2.2.7 / versionCode 42`。

### 42. 2.2.9 云端账号密码校验

- 云端同步从“直接查询账号名+密码生成的备份索引”升级为“账号注册表 + 密码校验”。
- 腾讯云 CloudBase 函数新增 `account-login`、`account-create`、`account-sync-put`、`account-sync-get`。
- 云端分别保存账号名查询 key 和密码校验值，不保存明文账号密码。
- 登录时如果云端已有同名账号但密码不匹配，会提示“该账号密码错误”，不会创建本地账号。
- 登录时如果云端没有该账号，会弹出“当前无账号，是否新建账号？”确认后再创建。
- 保留旧版 `sync-put/sync-get` 接口，并在新版登录时自动迁移 v2.2.8 的旧同步索引。
- 版本升级为 `2.2.9 / versionCode 44`，保证可以覆盖安装 `2.2.8 / versionCode 43`。

### 43. 2.2.10 账号入口 App 化

- “我的”页账号区默认改为身份卡片，只展示当前身份、记录数量和同步状态。
- 默认不再直接露出账号名、密码输入框，也不再展开管理员账户表单。
- 新增两个文本入口：“去登录”和“账号管理”。
- 点击“去登录”后进入登录账号面板，继续沿用云端账号校验和新建账号逻辑。
- 点击“账号管理”后进入管理员账户面板，保留管理密码、重置账号密码和删除账号能力。
- 版本升级为 `2.2.10 / versionCode 45`，保证可以覆盖安装 `2.2.9 / versionCode 44`。

### 44. 2.2.11 总览筛选与账号反馈修复

- 密码不匹配提示改用应用内确认弹窗，保持与恢复云端备份确认弹窗一致的交互风格。
- 管理员账户中只剩一个账号时，点击删除会提示“当前必须保留一个账号，无法删除”，不再无响应。
- 版本升级为 `2.2.11 / versionCode 46`，保证可以覆盖安装 `2.2.10 / versionCode 45`。
