# 秋招投递管理开发历史

本文档记录「秋招投递管理」从最初想法到当前 APK 版本的主要开发过程。整理依据包括项目对话需求、实际功能演进和 Git 提交历史。

当前版本：`2.0.1 / versionCode 27`

`2.x` 阶段从重要提醒能力开始。

## 迭代记录

### 01. 产品设想与核心设计

- 明确项目目标：做一款个人秋招投递状态管理软件，替代零散笔记。
- 确定核心记录字段：公司、来源、更新时间、状态、备注。
- 确定核心状态：待初筛、待面试、已拒绝。
- 初步确定桌面端优先，界面需要美观、顺滑、有工作台感。

### 02. 静态 HTML 原型

- 搭建第一版 `HTML / CSS / JS` 单页应用。
- 实现添加记录、查看记录、修改记录、删除记录。
- 使用浏览器 `localStorage` 做本地保存。
- 做出初版看板、列表和筛选体验。

### 03. 记录详情查看能力

- 修复添加记录后只能看到列表、不能查看每条详情的问题。
- 在统计卡片和状态入口中增加进入详情/列表的能力。
- 让用户可以从总投递、待初筛、待面试等入口查看对应记录。

### 04. Android APK 初版

- 将 Web 应用封装为 Android APK。
- 使用 Android WebView 运行本地前端资源。
- 形成可直接安装使用的移动端应用。
- 固定 APK 输出路径，方便后续反复安装测试。

### 05. 移动端导入导出

- 增加 JSON 导出能力，支持把本地记录备份出来。
- 增加 JSON 导入能力，支持手机和电脑之间迁移数据。
- 在 Android 端接入系统分享面板，方便分享到微信、文件、网盘等位置。
- 支持保存 `.json` 文件到手机本地。

### 06. 字段精简与重要程度

- 删除不必要的「子企业部门」字段。
- 删除「公众号名称」字段。
- 增加「重要程度」字段，支持 1-5 星。
- 根据重要程度调整卡片颜色：星级越高越偏红、越醒目；星级越低越偏绿色。

### 07. 状态统计与详情入口

- 强化「秋招进度工作台」下方各状态统计。
- 支持点击统计项后进入对应记录列表。
- 改善看板和列表之间的跳转体验。

### 08. 排序与统计视图

- 让看板和列表默认按重要程度从高到低排序。
- 增加统计视图，不再只有看板和列表。
- 统计中加入状态漏斗、来源分布、星级分布、高重要记录等分析模块。
- 让统计视图更贴近秋招复盘和推进节奏。

### 09. 本地账号隔离

- 增加本地账号功能。
- 不同账号默认隔离存储各自记录。
- 支持切换账号，避免多个身份或不同投递批次混在一起。
- 账号数据仍然保存在本机浏览器或 APK 本地存储中。

### 10. 上线为网页版 PWA

- 明确最简单上线方案：先作为静态网页/PWA 发布。
- 支持 GitHub Pages / Gitee Pages / Cloudflare Pages 等静态托管方式。
- 确认静态版本的限制：不同设备之间不会自动云同步。
- 完成 Git 仓库关联和远程推送流程。

### 11. 账号侧栏体验

- 将账号入口设计为点击左上角「秋」图标展开。
- 侧栏中展示当前账号、账号列表和账号管理操作。
- 避免使用 Android WebView 默认系统弹窗。
- 新增账号、重命名账号、删除确认等操作改为应用内界面。

### 12. 账号密码与管理密码

- 为账号增加本地密码保护。
- 切换已设密码账号时需要输入密码。
- 增加管理密码机制：通过管理密码后才能重置任意账号密码。
- 不允许查看旧密码，只允许重置，避免明文暴露。

### 13. 逾期预警机制

- 增加「更新时间超过 N 个月」的逾期预警。
- N 可以在界面中手动调整。
- 逾期记录高红色显示，用于提醒长期未更新的投递。
- 逾期记录在看板和列表中排在最后，内部再按星级排序。
- 统计视图中增加逾期数量栏目。

### 14. 今日待检查

- 增加「今日待检查」模块。
- 基于每条记录的下次检查日期筛选需要回看的投递。
- 增加「已检查」操作，点击后该记录会从今日待检查中移除。
- 已检查后下次检查时间顺延，减少重复提醒。

### 15. 城市字段与城市筛选

- 增加「城市」字段。
- 城市选择范围覆盖全国二线及以上城市。
- 杭州、北京、深圳、广州置于城市列表最前面。
- 其余城市按拼音顺序排列。
- 统计中增加城市筛选能力。

### 16. 多城市记录

- 将城市字段从单选升级为多选。
- 每条记录最多可选择 3 个城市。
- 城市筛选支持“包含该城市”的匹配逻辑。
- 修复城市筛选只能显示「全部城市」的问题。

### 17. 搜索跳转优化

- 搜索框输入后不再需要手动滑动到结果区域。
- 点击搜索图标或按回车后，自动跳转到符合条件的列表。
- 如果没有匹配结果，提示当前暂无该信息。
- 移除不美观的城市偏好主页面展示。

### 18. 作者标识与开源信息

- 在页面底部增加 `bai@copyright`。
- 增加「联系作者」入口。
- 点击后弹出作者信息弹窗。
- 展示开发者、联系邮箱和开源地址。

### 19. 自定义 APK 图标

- 设计并生成项目专属图标。
- 图标融合「秋」「招」「BAI」「投递卡片」「进度看板」等元素。
- 优化为偏苹果风格的毛玻璃、扁平化视觉。
- 替换 Android 默认图标。

### 20. 链接式导入导出

- 在 JSON 文件导入导出之外，增加链接式导入导出。
- 导出时可复制导入链接，方便通过微信、QQ、电脑浏览器传递。
- 导入时可以选择 JSON 文件，也可以手动粘贴链接。
- 取消自动检测 URL 导入，避免换账号或打开页面时莫名弹窗。

### 21. 云端短链接分享

- 接入 Cloudflare Workers + KV 免费云端方案。
- 导出时优先把当前账号数据上传到云端。
- 云端返回短链接，例如 `https://bai.a1623444091.workers.dev/i/Ab3x9K`。
- 导入时粘贴短链接，应用从云端读取 JSON 并导入。
- 如果云端不可用，保留本地长链接作为兜底。

### 22. 账号列表与记录卡片交互

- 多账号列表改为横向网格排列，一行从左到右排满后再换行。
- 新建账号时密码改为可选。
- 看板和列表中的记录都改成卡片式交互。
- 任意记录左滑可露出「置顶」和「删除」操作。
- 置顶记录优先展示；逾期预警记录仍按既定规则排在最后。

### 23. 管理员账户删除账号

- 取消当前账号区域的「删除当前账号」按钮。
- 当前账号区域恢复为两行：重命名账号、设置/更改账号密码。
- 将「本机密码管理」改名为「管理员账户」。
- 输入管理密码后，不仅可以重置所有账号密码，也可以删除任意账号。
- 删除账号需要二次确认，并且至少保留一个账号。

- 当前主要能力包括：本地账号隔离、管理员账户、投递看板、列表卡片、统计分析、城市筛选、逾期预警、今日待检查、JSON 导入导出、云端短链接导入导出、PWA 和 Android APK。

### 25. 2.0.0 重要提醒

- 版本号正式进入 `2.0.0 / versionCode 26`。
- 每条记录新增必填「重要提醒」字段。
- 重要提醒支持三种方式：无、选择具体截止日期时间、从当前时间开始计算未来 N 小时。
- 支持填写提醒备注，例如在线测评、笔试、确认面试时间等。
- 当重要提醒进入截止前 24 小时且尚未过期时，每次进入 App 都会弹出强提醒窗口。
- 强提醒窗口集中展示当前账号下需要处理的重要事项，并可点击进入对应记录详情。

### 26. 2.0.1 状态扩展与版本信息

- 状态体系扩展为 6 个：待初筛、待测评、待笔试、待面试、已拒绝、offer。
- 新增记录时，当前状态只展示待初筛和待笔试，减少误选。
- 编辑已有记录时，可以选择全部 6 个状态。
- 看板和列表卡片的快速流转区固定展示除待初筛外的 5 个状态，方便快速推进。
- 底部 `bai@copyright` 旁新增当前版本号和更新时间。
- 重要提醒默认值调整为「选择截止日期时间」，减少被忽略的概率。
